# Paper Trading Beta Review

Last updated: `2026-03-18`

This review combines code inspection with validation against the live beta DB at `C:\EquityTrackerData\portfolio.beta_research.db`.

## 1. Highest-Value Findings

### Finding 1: Training quiet-hours control is effectively bypassed

Code evidence:

- `equity_tracker/src/beta/services/training_service.py:620`
- `equity_tracker/src/beta/services/training_service.py:626`

Problem:

- `ensure_daily_training()` only skips when both conditions are true:
  - the training window is closed
  - there is no tracked core equity
- In practice there is tracked core equity, so the presence of one tracked core position disables the quiet-hours gate.

DB evidence:

- live settings still say `training_window_start_local = 22:00` and `training_window_end_local = 06:00`
- `beta_daily_training` completed successfully at `2026-03-18 07:52:35`, `07:24:56`, and `06:51:50`

Impact:

- the configured quiet-hours window is not authoritative
- the system can keep training in pre-open periods even when the settings imply it should stop
- this also increases model/version churn when no model is getting activated

Recommended fix:

- make the quiet-hours gate explicit and authoritative
- if tracked core equities need an exception, expose that as a separate setting rather than encoding it implicitly into the skip condition

### Finding 2: Off-hours job cadence is not restart-stable

Code evidence:

- `equity_tracker/src/beta/supervisor_process.py:682`
- `equity_tracker/src/beta/supervisor_process.py:690`
- `equity_tracker/src/beta/supervisor_process.py:692`

Problem:

- supervisor cadence state is stored only in memory
- `next_eod_bar_fetch_at`, `next_bar_backfill_at`, and the other timers are reset to `datetime.now(timezone.utc)` every time the supervisor starts

Impact:

- any supervisor restart outside market hours can immediately re-arm jobs that are intended to be daily or low-frequency
- this risks duplicate API spend and noisy job history for `beta_eod_bar_fetch`, `beta_intraday_bar_backfill`, and `beta_instrument_statistics_refresh`

Recommended fix:

- persist last-success timestamps per job in the DB and schedule from those timestamps
- treat restart as recovery, not as a full cadence reset

### Finding 3: Demo mode is enabled, but the path to actual demo trades is still blocked

Code evidence:

- `equity_tracker/src/beta/supervisor_process.py:513`
- recent shadow-cycle job details show `active_model_version = null` and `recommended = 0`

DB evidence:

- `beta_demo_positions = 0`
- `beta_signal_candidates = 78`, all `DISMISSED`
- all recent model versions are `CHALLENGER`
- no strategy versions are active

Impact:

- the system is operational in the sense of ingestion, training, scoring, and monitoring
- it is not operational in the sense of producing demo trades
- docs and UI should state this clearly so `demo_execution_enabled = 1` is not mistaken for actual trade generation

Recommended fix:

- document the current gating state in the UI and docs
- add an explicit "promotion blocked" or "no active strategy" status card
- consider reducing training churn until activation criteria or baseline policy are tuned

### Finding 4: The local watcher scripts were machine-specific

Original code evidence:

- `watch_signals.py:7`
- `watch_jobs.py:7`

Problem:

- both scripts hardcoded `C:\EquityTrackerData\portfolio.beta_research.db`
- they only worked on one machine layout and ignored `EQUITY_BETA_DB_PATH`

Action taken in this pass:

- replaced the hardcoded path with shared path resolution in `watch_beta_common.py`
- the scripts now resolve from `EQUITY_BETA_DB_PATH`, `EQUITY_DB_PATH`, repo-local data, or the existing external path
