### `equity_tracker/src/beta/services/intraday_bar_fetch_service.py`

**Change 1 — Fix `_usd_gbp_rate()` (was returning `None` for all USD instruments)**

The `fx_rates` table in the core DB is empty for USD/GBP. The old implementation queried it
and always returned `None`, causing all USD bars (IBM) to be silently dropped during GBP
conversion in `_upsert_bar`.

The fix derives the USD/GBP rate from `price_history` instead, by computing
`close_price_gbp / close_price_original_ccy` for the most recent USD record. The `fx_rates`
table is kept as a fallback.

```python
def _usd_gbp_rate() -> Decimal | None:
    # Primary: derive from price_history where FX conversion has been applied
    try:
        from ...db.models import PriceHistory
        from ..core_access import core_read_session
        with core_read_session() as sess:
            row = sess.scalar(
                select(PriceHistory)
                .where(
                    PriceHistory.currency == "USD",
                    PriceHistory.close_price_gbp.is_not(None),
                    PriceHistory.close_price_original_ccy.is_not(None),
                )
                .order_by(desc(PriceHistory.price_date))
                .limit(1)
            )
            if row is not None:
                gbp = Decimal(str(row.close_price_gbp))
                native = Decimal(str(row.close_price_original_ccy))
                if native > 0 and gbp > 0:
                    return (gbp / native).quantize(Decimal("0.000001"))
    except Exception:
        pass
    # Fallback: fx_rates table
    try:
        from ...db.models import FxRate
        from ..core_access import core_read_session
        with core_read_session() as sess:
            row = sess.scalar(
                select(FxRate)
                .where(FxRate.base_currency == "USD", FxRate.quote_currency == "GBP")
                .order_by(desc(FxRate.rate_date))
                .limit(1)
            )
            if row is not None and row.rate:
                return Decimal(str(row.rate))
    except Exception:
        pass
    return None
```

**Change 2 — Add `backfill_historical_bars()` method**

New static method that fetches historical 1-minute bars for HELD and ACTIVE_THESIS instruments,
working backwards in 5000-bar pages until `target_days` of history is covered or the credits
budget is exhausted. Uses `source = "twelvedata_1min_historical"`. Upsert is idempotent — safe
to run repeatedly. Skips instruments that already have history back to the target window.

**Change 3 — Extract `_fetch_bars_with_params()` helper**

`_fetch_bars()` now delegates to `_fetch_bars_with_params(params: dict)`, which allows
`backfill_historical_bars()` to pass arbitrary params (including `end_date`) without
duplicating the httpx call and error-handling logic.

Inserted between EOD bar fetch and statistics refresh:

```python
# Historical bar backfill — daily cadence, outside market hours
if settings.intraday_bar_fetch_enabled and settings.intraday_bar_backfill_enabled \
        and now >= next_bar_backfill_at and not market_open_light_mode:
    _run_job(
        job_name="beta_intraday_bar_backfill",
        job_type="intraday_execution",
        op=lambda: BetaIntradayBarFetchService.backfill_historical_bars(
            priority_items=list(
                BetaIntradayPriorityService.build_watchlist(settings, now_utc=now)["items"]
            ),
            target_days=settings.intraday_bar_backfill_target_days,
            credits_budget=settings.intraday_bar_backfill_credits_budget,
        ),
    )
    next_bar_backfill_at = now + timedelta(hours=24)
```

Cadence: 24 hours. Guarded by `not market_open_light_mode` — only runs outside market hours
to avoid consuming credits during live trading windows.

## Supervisor Restart Issue

**The supervisor is currently stopped.** The problem is that the supervisor is being started
from within a git bash tool session, and when that tool session ends its background processes
are killed (even with `nohup`).

The correct way to restart the supervisor is via the web application, which uses
`BetaRuntimeManager` with `subprocess.Popen(creationflags=DETACHED_PROCESS)` to spawn a
properly detached Windows process. The supervisor was originally started this way (PID 18980).

**To restore normal operation:**

1. Start the web app as normal (e.g. `uvicorn` command in a terminal)
2. The web app's `auto_start_supervisor = True` setting will re-launch the supervisor
   automatically on startup
3. Alternatively, use the beta control panel in the web UI to start the supervisor manually

**The code changes are all correct and will take effect as soon as the supervisor next starts.**
